    Grade: üö® CRITICAL

    The Real Story¶

    The local go.mod dependency is NOT just a development convenience‚Äîit's a required patch for a critical bug in HashiCorp's go-plugin library.

    From LLM_HANDOFF.md:

    ### 1. **go-plugin Bug: TLSProvider Certificate Not Extracted**
- Location: /Users/tim/code/tf/go-plugin/server.go (upstream)
- Issue: When using custom TLSProvider, the server certificate
  wasn't being extracted into the go-plugin handshake
- Impact: Clients received empty serverCert field in handshake,
  causing TLS failures
- Fix Applied: Patched /Users/tim/code/gh/hashicorp/go-plugin/server.go

    Technical Details of the Patch¶

    Location: go-plugin/server.go (lines ~303-311)

    The Fix: 

    if tlsConfig != nil && len(tlsConfig.Certificates) > 0 {
    serverCert = base64.RawStdEncoding.EncodeToString(
        tlsConfig.Certificates[0].Certificate[0]
    )
}

    What it does: Extracts the server certificate from custom TLSProvider and includes it in the handshake protocol (field 6).

    Without this fix: All custom elliptic curve TLS configurations fail.

    3. Known Architectural Limitations¶

    3.1 Python Client ‚Üí Go Server Unsupported¶

    Grade: ‚ö†Ô∏è SIGNIFICANT LIMITATION

    The Issue¶

    From LLM_HANDOFF.md:

    **Supported language combinations:**
- ‚úÖ Python client ‚Üí Python server (KVClient)
- ‚úÖ Go client ‚Üí Go server (soup-go)
- ‚úÖ Go client ‚Üí Python server (soup-go)
- ‚ùå Python client ‚Üí Go server (NOT supported by pyvider-rpcplugin -
     times out on handshake)

    Root Cause¶

    pyvider-rpcplugin limitation: The Python RPC client library (pyvider-rpcplugin) has a bug or incompatibility when connecting to Go servers spawned via go-plugin.

    Symptoms: - Connection timeout on handshake - No error message, just timeout - Works fine for Python ‚Üí Python

    Impact: - 40 tests skipped - Cannot test full matrix - Limits real-world use cases

    3.2 Impact Analysis¶

    For Development¶

    Test Coverage: - Can test Python server implementation - Can test Go server implementation - Cannot test Python client against Go server - Cannot validate cross-language client compatibility

    Workaround: - Use Go client to test Go server - Document limitation - Mark tests as skipped

    For Production Use¶

    Deployment scenarios:

    Client Server Status Notes
    Python provider Python RPC service ‚úÖ Works Fully supported
    Go provider Go RPC service ‚úÖ Works Fully supported
    Go provider Python RPC service ‚úÖ Works Supported
    Python provider Go RPC service ‚ùå Blocked NOT SUPPORTED

    Real-world impact: - If users want to write Python Terraform providers that use Go RPC backends, they cannot use tofusoup's patterns - This is a niche but valid use case

    3.3 Resolution Options¶

    Option A: Fix pyvider-rpcplugin ‚úÖ RECOMMENDED¶

    Approach: 1. Debug Python client ‚Üí Go server handshake 2. Identify timeout/protocol mismatch 3. Fix in pyvider-rpcplugin 4. Release updated pyvider-rpcplugin 5. Update tofusoup dependencies

    Investigation areas: - Handshake protocol compatibility - Certificate parsing differences - Timeout configuration - gRPC version mismatches

    Timeline: 1-3 weeks (investigation + fix)

    Option B: Document Limitation ‚ö†Ô∏è INTERIM¶

    Approach: - Add prominent note in README - Update API documentation - Mark tests as "known limitation" - Provide workaround examples

    Already done: Tests are skipped, but not well-documented

    Still needed: - User-facing documentation - Workaround patterns - Alternative approaches

    Option C: Alternative RPC Framework üî¥ MAJOR REFACTOR¶

    Approach: - Switch from go-plugin to alternative - Options: gRPC-gateway, custom RPC, hashicorp/yamux directly

    Pros: - Could fix limitation - More control

    Cons: - Massive refactor - Breaks compatibility - Loses go-plugin benefits

    Recommendation: Only if Option A proves impossible

    3.4 Recommended Approach¶

    Immediate (Week 1): - Implement Option B: Document limitation clearly - Add to README.md, ARCHITECTURE.md, troubleshooting.md - Create GitHub issue tracking the limitation

    Short-term (Week 2-4): - Implement Option A: Debug and fix pyvider-rpcplugin - Create minimal reproduction case - Engage pyvider maintainers (if external) or fix internally
